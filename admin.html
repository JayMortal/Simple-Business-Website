🐳 How to Update (Docker) / Docker 更新方法

This project runs in Docker. Updates require SSH access to run the commands below. Your site data (site-data.json) is preserved via volume mount.

SSH Update Command
cd /opt/1panel/apps/Simple-Business-Websitebuilder
git pull
docker compose down
docker compose build --no-cache
docker compose up -d
docker image prune -f
⚠ 如果更新后页面没有变化（Cloudflare 用户）

Log in to Cloudflare → your domain → Caching → Purge Everything. To prevent this in future, add a Cache Rule to Bypass cache for *.js and *.css.

v0.2 — Backend rewrite / 后端重构
  • Backend migrated from PHP to Node.js + Express / 后端从 PHP 迁移至 Node.js + Express
  • Auth replaced with express-session httpOnly cookie / 认证改为 express-session httpOnly Cookie，彻底解决同步失败问题
  • HTML served with Cache-Control: no-cache — no more stale pages after update / 服务器对 HTML 强制 no-cache，更新后浏览器立即加载新版本
  • Admin panel auto-detects browser language on first visit / 管理后台首次访问自动检测浏览器语言，英文用户不再看到中文
  • Docker volume now mounts only data/ — no chown workarounds needed / Docker volume 只挂载 data/ 目录，无需 chown 权限修复
  • Environment config via .env file (ADMIN_PASSWORD, SESSION_SECRET) / 通过 .env 文件配置初始密码和 Session 密钥
